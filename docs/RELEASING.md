## Перед отправкой на GitHub

- [ ] `python scripts/check_release_privacy.py` завершился успешно.
- [ ] `python -m unittest discover -s tests -v` завершился успешно.
- [ ] В `git status --short` нет неожиданных файлов.
- [ ] В скриншотах и GIF видны только синтетические данные.
- [ ] Папки `logs/`, `.venv/`, `build/` и `dist/` отсутствуют в архиве.
- [ ] Для проекта выбрана и добавлена подходящая лицензия.

## Windows EXE

Готовый `.exe` лучше прикреплять к **GitHub Releases**, а не коммитить в основную ветку. Так исходный репозиторий остаётся лёгким, а пользователю доступна понятная версия сборки и контрольная сумма.

Перед публикацией сборки:

- проверьте запуск на чистой Windows без установленного Python;
- откройте несжатый DICOM, JPEG‑LS и JPEG 2000;
- проверьте встроенные Inter и Lucide‑ресурсы;
- убедитесь, что рядом со сборкой нет тестовых исследований и логов;
- приложите SHA‑256 контрольную сумму файла;
- укажите версию, дату и список изменений.
